# AcuGuide Competition Demo Script

## 展示前準備

1. 使用 Chrome 或 Edge 開啟 `http://127.0.0.1:5173/`。
2. 確認投影解析度可用，建議 1920 x 1080。
3. 若要展示相機 AR，允許瀏覽器相機權限；若現場相機不可用，直接展示備援訊息與 3D/文字完成流程。
4. 不需要網路即可展示主要流程；Gemma 大模型不在 repo 內，現場以本機 RAG preview fallback 展示。

## 開場說法

AcuGuide 的目標不是做另一個靜態穴位圖鑑，而是把穴位知識轉成個人身體上的定位引導。Web Demo 展示的是 iOS App 企劃中的核心互動：安全審核、穴位推薦、3D 空間理解、相機輔助定位、按需校正與完成回饋。

## 流程一：臉部美容

1. 首頁保留在「臉部美容」。
2. 選擇「改善眼睛疲勞」或在輸入框輸入 `黑眼圈`。
3. 點選「取得推薦」。
4. 說明推薦卡目前是本機 RAG preview；若 Gemma 模型不存在會自動備援。
5. 點選「確認推薦」。
6. 展示臉部 AR Demo，若相機可用可點「開啟鏡頭」。
7. 點「完成」，在完成頁儲存一筆回饋。

## 流程二：肩頸痠痛

1. 點選「重新開始」。
2. 選擇「身體部位疼痛」。
3. 選擇「肩頸痠痛」或輸入 `肩頸痠痛` 後點「取得推薦」。
4. 旋轉或點選 3D 人體模型上的穴位標記。
5. 點「確認推薦」進入身體 AR Demo。
6. 說明 A-D 等級：A 可即時追蹤，B 為關鍵點推算，C 需校正，D 僅作 3D 參考。
7. 點「完成」。

## 流程三：脹氣與校正

1. 點選「重新開始」。
2. 選擇「調理身體」。
3. 輸入 `飯後脹氣 腹部悶`。
4. 點「取得推薦」後說明安全審核已先通過，接著才顯示 RAG preview。
5. 點「確認推薦」。
6. 點「校正」進入手部比例尺頁面。
7. 若相機不可用，使用滑桿作為手動比例尺備援。
8. 點「完成校正」回到指引，再點「完成」。

## 流程四：高風險症狀攔截

1. 回到首頁。
2. 輸入 `急性胸痛 呼吸困難`。
3. 點「取得推薦」。
4. 展示安全攔截卡，指出本次不顯示一般穴位推薦，也不進入 Gemma 推薦。

## 流程五：相機拒絕備援

1. 在瀏覽器封鎖相機權限，或使用測試環境覆寫相機為拒絕。
2. 進入任一 AR Demo 後點「開啟鏡頭」。
3. 展示「已切換為 3D 與文字備援 Demo」訊息。
4. 使用右側模型與文字仍可完成流程。

## 收尾說法

Web Demo 已完成競賽現場需要的主要操作閉環。正式 iOS 版本會把 Web 中的 3D/MediaPipe proof-of-concept 升級為 ARKit、Vision、LiDAR 或 TrueDepth，並接上經專家審核的完整知識庫與裝置端 Gemma 推論。
